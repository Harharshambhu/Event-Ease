Module 1: Credentials — pass & access management
Phase 1 — Configuration (admin, pre-event)
ElementsTasksObjectives
Credential typese.g., All-access, Backstage, Vendor, Media, VIP, Artist, Volunteer Define each credential type with name, access level, and visual design (background color, printed fields) Establish a clear access hierarchy so security knows exactly what each pass permits on sight
Badge design templatesPer-type customizable layout Configure which variable data prints on each type (company name vs. volunteer committee vs. artist name) Produce on-brand, instantly recognizable badges that encode role/access at a glance
Approval workflowsSingle or multi-level, conditional Set up approval steps — who approves vendor credentials vs. media vs. VIP. Configure conditional logic (e.g., all-access requires director approval) Prevent unauthorized access by enforcing sign-off chains before any credential is issued
Item blocksRequestable credential packages Create item blocks with specific credential types, quantities, price per item, and limits. Assign blocks to group types or individual groups Control what each stakeholder type can request — vendors get vendor passes, not all-access
Barcode / RFID integrationTicketing partner sync Import or generate barcodes. Configure integration with ticketing partners (Eventbrite, Intellitix). Map barcode pools to credential types Enable instant onsite verification — scan a wristband and know exactly who they are and what they can access
Phase 2 — Collection (stakeholders, via portal or form)
ElementsTasksObjectives
High-level count requestsPhase 1 of credentialing Group primary contact submits how many credentials they need per type (e.g., "We need 12 vendor passes and 3 backstage") Get early headcount estimates for production planning before individual names are finalized
Personnel name submissionPhase 2 of credentialing Group contact enters individual names, roles, and credential type per person through portal. Can add, edit, remove names up to deadline Collect accurate person-level data for badge printing, security lists, and onsite verification
Portal request formsCustom fields per group type Stakeholder fills out required fields per person: name, title, photo upload, emergency contact, dietary info, credential type needed Gather all credential-adjacent data in one submission rather than chasing across emails
Standalone formsPublic link, no portal needed Share a direct form URL for one-off credential requests (e.g., late additions, day-of walk-ins) Handle exceptions without breaking the portal workflow — capture data from anyone, anywhere
Manual + bulk importAdmin-side data entry Admin manually adds credentials for VIPs/executives or imports CSV for large batches (e.g., 200 volunteer names) Fill gaps where stakeholders don't self-serve — ensure no one falls through the cracks
Phase 3 — Review & approval (admin)
ElementsTasksObjectives
Approval queuePending requests dashboard Review incoming credential requests. Approve, deny, or request changes per person. Add notes explaining denial reason Maintain security standards while processing high volumes efficiently
Status reportsReal-time completion tracking Monitor which groups have submitted, which are overdue, which have pending approvals. Dashboard flags "next actions needed" Proactively identify bottlenecks — don't wait until day-of to discover 8 vendors never submitted
Automated remindersEmail nudges for overdue groups Configure automatic reminder emails to groups who haven't submitted by deadline. Customize message and frequency Reduce manual chasing — let the system nag so the admin doesn't have to
Barcode assignmentAuto-assign on approval When admin approves a credential, the next available barcode from the pool auto-attaches to that person's order Eliminate manual barcode tracking spreadsheets — one approval action cascades to issuance-ready
Phase 4 — Issuance & onsite (event day)
ElementsTasksObjectives
Badge printingPre-event or onsite Print badges per credential type with correct design, variable data, and barcode. Take onsite photos for new/walk-in attendees Produce professional, secure, scannable credentials at scale
Check-in appMobile scanning stations Scan barcode/RFID wristband at entry points. App verifies against approved credentials in real time. Logs check-in timestamp Validate identity and access level in under 3 seconds — no paper lists, no guessing
Pickup trackingWho collected their badge Mark credentials as picked up at will-call or box office. Track outstanding uncollected badges Know exactly who is onsite and who hasn't arrived — critical for headcounts and security
Portal status visibilityStakeholder self-check Stakeholder views their portal to see each person's credential status: approved, pending, denied, issued, checked in Eliminate "is my team's credentials ready?" phone calls — self-service status reduces admin load
