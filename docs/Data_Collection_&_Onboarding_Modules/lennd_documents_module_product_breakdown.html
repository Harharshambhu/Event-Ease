Module 3: Documents & files — two-way file exchange
Unlike Forms (structured data input), Documents is about file exchange — both requesting files FROM stakeholders (COIs, contracts, riders) and sharing files TO them (maps, production guides, training videos). It's the "Dropbox replacement" layer.
Phase 1 — Request configuration (admin)
ElementsTasksObjectives
Document request typesNamed categories of files needed Create document request types: Certificate of Insurance (COI), W-9, signed contract, tech rider, stage plot, logo files, vehicle registration, liquor license — any named document category Name each document clearly so stakeholders know exactly what file you need — "COI" not "please send us your paperwork"
Sample file attachmentsReference examples for stakeholders Attach a sample or template file to each document request: "Here's a sample COI format we accept" or "Use this contract template." Stakeholder downloads the reference before uploading their version Reduce back-and-forth — stakeholders submit correctly formatted documents on the first attempt because they saw the example
Assignment to group typesWhich types need which docs Assign document requests to portals per group type: vendors need COI + W-9 + contract; artists need tech rider + stage plot; volunteers need signed waiver. Different requirements per type Each group type sees only their relevant document requests — volunteers aren't confused by a COI request meant for vendors
Individual group assignmentsOverride for specific groups Add document requests to a specific group beyond the type-level defaults: one vendor needs a special permit, one artist needs a licensing agreement. Per-group customization Handle edge cases without creating new group types — add a one-off document request to a single vendor's portal
Due dates per documentStaggered deadlines Set different due dates per document type: COI due 30 days before event, tech rider due 14 days before, contract due 60 days before. Overdue items flagged automatically Stagger collection to match the event's operational timeline — insurance needed for contracts, riders needed for production planning
Phase 2 — Stakeholder file submission & access
ElementsTasksObjectives
Document upload via portalDrag-and-drop file submission Stakeholder sees their required documents in the portal: document name, description, sample file, due date, upload button. Upload their file to fulfill the request Clear, guided file submission — the stakeholder knows what to upload, has a reference example, and sees the deadline in the same view
Submission status visibilityUploaded vs. pending vs. overdue Portal shows each document's status: not submitted, uploaded (awaiting review), accepted, or rejected with admin notes. Stakeholder tracks their own compliance Self-service status checking — stakeholder doesn't need to email "did you get our COI?" — they see it's been accepted in the portal
Re-upload / versioningReplace rejected documents If admin rejects a document (wrong format, expired, missing information), stakeholder can re-upload a corrected version. Previous version remains for reference Handle the review-reject-resubmit cycle within the portal instead of an email thread with 5 attachments
Admin-shared files (inbound)Files shared TO stakeholders Admin uploads files to the portal's Files/Event Info section: venue maps, parking guides, production schedules, safety protocols, training videos. Stakeholders view and download Two-way exchange in one place — the stakeholder sends you their COI and finds your venue map in the same portal
Phase 3 — Review, tracking & management (admin)
ElementsTasksObjectives
Document tracking dashboardCross-group submission status View all groups by document type: how many have submitted COIs, how many are overdue, how many need re-submission. Filter by document type, group type, status One screen answers "do we have all the insurance certificates?" — don't wait until contract signing to discover 12 vendors haven't sent COIs
Document reviewAccept or reject per file Open submitted documents, verify contents, mark as accepted or rejected. Rejection includes a note explaining what needs to change. Accepted documents are locked Quality gate on every document — prevent an expired COI or an unsigned contract from slipping through
Automated remindersSystem nudges for missing docs Automatic email reminders to groups with overdue or unsubmitted documents. Configurable timing and message per document type Persistent, consistent follow-up without admin effort — the system reminds weekly until the document is submitted
Bulk exportDownload all files per type Export all submitted documents of a given type (e.g., download all 47 vendor COIs as a zip). Export metadata as CSV for compliance records Hand off complete document packages to legal, insurance, or compliance departments — not a folder of 200 emails with attachments
