Module 2: Forms — custom data collection engine
Phase 1 — Form building (admin)
ElementsTasksObjectives
Field builderCustom question types Build forms using Lennd's field types: text, number, dropdown, multi-select, date, file upload, phone, email, address, long text, checkbox. Each field is a reusable CRM "field" — create once, use across multiple forms Collect exactly the data you need in the format you need it — structured fields, not free-text emails that need manual parsing
Conditional logicShow/hide fields based on answers Configure field visibility based on previous responses: if "Credential type = All-access" then show "Justification reason" field. Fields appear/disappear dynamically Keep forms short and relevant — vendors don't see questions meant for media, even on a shared form template
Required vs. optional fieldsSubmission validation Mark fields as required or optional. Form won't submit until all required fields are filled. Validation rules per field type (email format, phone format) Prevent incomplete submissions that require follow-up — get complete data on the first pass
Form assignmentTo portals, group types, or individuals Assign form to a group type portal (all vendors get it), to a specific group (only Acme AV gets this custom form), or publish as standalone URL for public access Flexible distribution — same form engine serves portal-embedded and standalone use cases
Due datesPer form, per assignment Set submission deadlines per form. Different deadlines possible per group. Overdue forms flagged in admin dashboard and stakeholder portal Enforce timeline without manual chasing — the system tracks who's late, not the admin's memory
Reusable field libraryFields persist across forms and events Fields created in one form are available to reuse in other forms and future events. "Company name," "Primary contact email," "T-shirt size" — define once, reuse everywhere Eliminate rebuilding the same questions each event — the field library grows over time and speeds up form creation
Phase 2 — Distribution & collection (stakeholders)
ElementsTasksObjectives
Portal-embedded formsWithin the stakeholder's portal Stakeholder sees assigned forms in their portal dashboard under "Forms" section. Incomplete forms show as action items with due dates. Clicks to open and fill Forms live alongside credentials, assets, catering — the stakeholder doesn't switch tools or URLs to complete different tasks
Standalone form linksPublic URL, no portal needed Share a direct form URL for one-off respondents: ticket request forms, volunteer applications, media accreditation. Anyone can access — no account needed Lowest-friction collection path — for stakeholders who don't need an ongoing portal relationship, just a single form submission
Form editing after submissionPortal forms are dynamic Unlike standalone forms (submit once), portal forms can be revisited and edited by the stakeholder until the close date. Changes update the CRM in real time Accommodate evolving information — vendor's load-in time changes, artist's rider is updated, contact person swaps
File upload fieldsAttach documents within forms Form fields that accept file uploads: logos, insurance certificates, signed contracts, tech riders. Uploaded files attach to the group's record in the CRM Collect structured data AND supporting documents in one submission — no separate email needed for the attachment
CRM auto-populationData flows into central database Form responses automatically populate the group and person records in the event CRM. "Company name" from the form becomes the group name in the database. No manual data entry The form is the data entry interface for the CRM — stakeholders unknowingly populate the admin's operational database
Phase 3 — Review & management (admin)
ElementsTasksObjectives
Submission trackingPer-form, per-group status Dashboard shows which groups have submitted each form, which are incomplete, which haven't started. Filterable by form, group type, due date status At-a-glance compliance visibility — know that 38 of 47 vendors submitted their insurance form, 9 are outstanding
Response reviewView individual submissions Open any group's form responses. Review answers, view uploaded files, check for completeness. Can leave internal notes on responses Quality check submitted data before it drives downstream decisions (credential printing, catering orders, asset allocation)
Bulk exportCSV/Excel of all responses Export all form responses for a given form across all groups. One row per group, one column per field. Ready for analysis or handoff to other departments Bridge the gap between Lennd and external systems — export data for the catering vendor, the print shop, the security team
Automated remindersNudge non-submitters System sends automatic email reminders to groups with incomplete or unsubmitted forms. Configurable message, frequency, and trigger conditions Reduce admin follow-up burden — the system nags consistently and tirelessly; the admin focuses on approvals, not chasing
