Module: Check-in app — onsite credential & catering verification
The check-in app is a mobile application used by onsite staff — not by stakeholders. It's the physical enforcement layer that validates the digital records from the Credentials and Catering modules against real people at real entry points.
Phase 1 — Pre-event setup (admin)
ElementsTasksObjectives
Scan station accountsOne user per entry point Create individual user accounts for each scanning station (Gate A, Gate B, VIP entrance, catering tent, backstage). Assign each station user to a permission profile that includes check-in access Track which station processed which scan — critical for post-event analysis ("Gate B was 3x slower than Gate A") and accountability
Barcode/RFID syncCredential-to-scan mapping Ensure all approved credentials have assigned barcodes/RFID tags. Verify ticketing partner integration is live (Eventbrite, Intellitix). Test scan-to-lookup response time Guarantee that every approved person in the system has a scannable identifier — no orphaned credentials that can't be verified
Check-in rulesAccess control logic Configure what happens when a credential is scanned: which access zones it permits, whether re-entry is allowed, whether time restrictions apply (e.g., load-in passes only valid before noon) Encode the access policy into the scan logic — the app enforces rules that gate staff don't need to memorize
Catering check-in configMeal scanning rules Configure which meals can be scanned at which stations. Set manager override code for exceptions. Define whether one meal at a time or bulk meal claims are allowed Prevent double-scanning (person claims lunch twice) and unauthorized meal access while allowing manager overrides for exceptions
Device preparationApp installation + testing Install Lennd check-in app on designated devices (tablets, phones). Log in with station account. Test scan speed, connectivity, and data sync. Verify fallback for offline scenarios Ensure every scanning station works flawlessly before the first person arrives — tech failures at gates cause cascading delays
Phase 2 — Credential check-in (onsite, entry points)
ElementsTasksObjectives
Barcode / QR scanningCamera or external scanner Staff scans person's credential barcode or QR code using device camera or connected hardware scanner (Socket Mobile, Linea Pro). App looks up the person record in real time Validate identity and access level in under 3 seconds — fast enough to keep entry lines moving at high volume
RFID wristband scanningContactless tap verification For RFID-enabled events, person taps wristband against reader. App matches wristband ID to person record and credential order. No visual scan needed — tap and go Enable the fastest possible check-in for high-throughput entry points (festival gates, arena entrances with thousands arriving simultaneously)
Name search fallbackFor unreadable or missing credentials If barcode won't scan or person forgot their badge, staff searches by name in the app. Pulls up person record with photo (if uploaded), credential type, and access level Never block entry due to a tech failure — always have a manual verification path so no one is stranded at the gate
Check-in confirmationVisual feedback on scan On successful scan: green confirmation with person's name, credential type, and access zone. On denied scan: red alert with denial reason (expired, wrong zone, already checked in, not approved) Give gate staff an instant, unambiguous go/no-go signal — no interpretation needed, no judgment calls at the barrier
Duplicate scan detectionAnti-fraud safeguard If a credential has already been checked in, the app flags it: "Already checked in at [Gate A] at [2:15 PM]." Staff can override with reason or deny entry Prevent credential sharing (one person lends badge to another after entry) and catch cloned/forged barcodes
Check-in timestamp logPer-person arrival record Every scan records: person name, credential type, station ID, timestamp, scan result (admitted / denied / override). Stored in real time to admin dashboard Build a complete arrival record — know who is onsite, when they arrived, and through which gate. Essential for emergency headcounts
Phase 3 — Catering check-in (onsite, meal stations)
ElementsTasksObjectives
Meal scan via wristbandPerson's pass → meal lookup Person scans their wristband/badge at catering station. App finds their person record, locates approved meals for the current meal period, and claims one meal Only people with approved meals eat — eliminate unauthorized meals that inflate catering costs by 15-30% at typical events
Group-level meal claimSearch by group name If person doesn't have individual meal assignment, staff searches their group name. App shows available group meal pool and claims one against the group's allocation Handle the common case where meals are allocated to "Acme AV Co. — 12 lunches" rather than to individual names
Manager overridePassword-protected exception When a person has no approved meal (new arrival, last-minute addition, admin oversight), catering manager enters override code. Meal is granted, logged against the wristband ID, and flagged as override Never turn someone away hungry due to a data gap — but track every exception so the admin knows where the system leaked
One-meal-at-a-time enforcementPer-scan claim limit App claims one meal per scan. Person cannot scan again for the same meal period. Prevents double-dipping. Separate claim for breakfast vs. lunch vs. dinner Strict 1:1 meal tracking — the single biggest lever for controlling catering spend at large events
Dietary flag displayPerson's restrictions on scan When person scans in, app shows their dietary restrictions (vegetarian, nut allergy, etc.) to catering staff alongside the meal claim confirmation Catering staff can direct people to the correct food line immediately — no asking, no guessing, no allergen accidents
Phase 4 — Real-time monitoring & post-event (admin)
ElementsTasksObjectives
Live attendance dashboardReal-time onsite headcount Admin dashboard shows live count of checked-in people by credential type, gate, and time. Updates with every scan across all stations Answer "how many people are onsite right now?" at any moment — critical for security, fire marshal compliance, and capacity management
Live catering usageMeals claimed vs. remaining Dashboard shows meals claimed vs. approved vs. unused for current meal period. Alerts if usage approaches capacity or if a catering station is unusually slow Adjust catering operations in real time — open an additional station, extend a meal period, or reduce tomorrow's order based on today's actual usage
Check-in log exportComplete scan history Export full scan log: every check-in and meal scan with person, credential, station, timestamp, result. CSV/Excel format for post-event analysis Post-event audit trail — "Vendor X says their team was there by 6 AM" can be verified against scan timestamps
No-show identificationApproved but never checked in Cross-reference approved credentials against check-in logs to identify people who were credentialed but never arrived Optimize next year's credential allocation — if 20% of a vendor's credentials go unused, reduce their allocation and free passes for others
