Module: Event CRM — the central contact & data backbone
The CRM is not a standalone module users interact with directly — it's the data layer beneath every other module. Credentials, catering, assets, guest lists, portals, forms, payments — all read from and write to the CRM. When a vendor submits a credential request through their portal, the data lands in the CRM. When an admin approves it, the CRM record updates. Every module is a view into this database.
Data architecture — the two-entity model
ElementsTasksObjectives
GroupsOrganizations / companies / teams Each group is an entity participating in the event: "Acme AV Co." (vendor), "RedBull" (sponsor), "Gate A Volunteers" (volunteer team). Groups have a primary contact, a type, and contain people. Groups hold aggregate data: total credential count, asset orders, catering allocation, payment balance Model the real-world structure — agencies deal with organizations, not loose individuals. The group is the contracting entity, the billing target, and the accountability unit
PeopleIndividuals within groups Each person belongs to one or more groups. Stores: name, email, phone, photo, title, dietary restrictions, emergency contact, and any custom fields. Each person has individual credential orders, catering assignments, and check-in records Track individuals for badge printing, meal scanning, and access verification — the group gets 12 passes, but each of the 12 people needs a name on a badge and a barcode to scan
Group typesVendor, Artist, Sponsor, Volunteer, etc. Define types during event setup. Use preset types (Vendor, Artist, Sponsor, Media, Volunteer, Speaker, Exhibitor, Staff, Athlete) or create custom types. Each type determines portal configuration, form assignments, and module access. A "Default" catch-all type can be created for unsorted contacts during import Enable batch configuration — assign one portal template to all vendors at once, instead of configuring 47 individually. Types are the segmentation layer that makes modular portals possible
People typesMirror or extend group types People types can mirror group types (a person in a vendor group = vendor person) or be independent (a "VIP" person type across multiple group types). Used for filtering, reporting, and credential differentiation Enable person-level segmentation beyond their group — a person can be tagged as "Day-of Hire" or "Returning Staff" regardless of which group they belong to
Field management — the custom data schema
ElementsTasksObjectives
Pre-populated fieldsCommon contact fields out-of-box Lennd ships with standard fields for groups (company name, address, phone, website, primary contact) and people (first name, last name, email, phone, title, photo, dietary restrictions). Ready to use without configuration Eliminate setup time for common data — the 80% of fields every event needs are already there
Custom fieldsEvent-specific data capture Create custom fields of any type: text, number, dropdown, checkbox, date, file upload, photo upload, multi-select, URL. Add descriptions to guide data entry (e.g., "Upload a photo of a recent booth you have hosted"). Fields are reusable across forms and portals Capture any event-specific data the admin needs — T-shirt sizes, arrival times, vehicle license plates, insurance policy numbers, social media handles — without workarounds
Field groupsOrganized categories of fields Organize fields into logical groups: "Contact Information," "Compliance Documents," "Year-Specific Questions," "Group-Type-Specific Questions." Especially useful when field count grows large across years Keep the CRM schema manageable as events grow complex — separate 2025-specific questions from 2026-specific questions, vendor questions from artist questions
Lookup fields (form-to-CRM link)Bidirectional data binding When a CRM field is added to a form as a question, it functions as a lookup — form responses are linked to the contact's CRM profile. New form submissions or profile changes update the data across the entire platform automatically Single source of truth — a dietary restriction updated via portal form instantly reflects in catering reports, badge printing, and the person's CRM record. No manual sync
Record layout per typeCustom profile view per group/people type Configure which fields display when viewing a contact profile, per type. Vendor profile shows: COI status, asset orders, load-in time. Artist profile shows: rider status, performance schedule, guest list. Different layouts for different roles When an admin clicks on a vendor vs. an artist, they see immediately relevant information — not every field from every type dumped into one page
Contact management — adding, importing, maintaining
ElementsTasksObjectives
Manual contact creationAdd one group or person at a time Click "Add Group" or "Add Person" in CRM. Fill in required fields (name, type, primary contact email). Optionally grant portal login access during creation. Assign person to one or more groups Quick entry for one-off additions — a new vendor confirmed yesterday, a VIP guest added last minute
CSV/Excel importBulk contact loading Upload CSV/XLS file with contact data. Map Lennd fields to column headers (mandatory fields auto-selected). Set default values for unmapped columns. Import into Groups or People. Pro tip: import people into Groups view to auto-associate Group Name Handle the "200 vendor contacts from last year's spreadsheet" scenario — go from spreadsheet to fully populated CRM in minutes instead of hours of manual entry
Intake form auto-creationContacts created from form submissions When an external party fills out an onboarding form or application, a new group/person record auto-creates in the CRM. Form responses populate the corresponding CRM fields. No admin intervention needed Self-service contact creation — the vendor onboards themselves, and the CRM populates automatically. The admin never manually types a vendor's company name
Bulk editMass update across selected contacts Select multiple groups or people in the CRM view. Apply bulk changes: update type, change a field value, assign to a portal, add a tag. Filters + bulk edit = targeted mass operations Handle year-over-year housekeeping efficiently — "move all 2025 vendors to 2026 vendor type" or "update all volunteer T-shirt sizes to this year's options"
ExportCSV/Excel data extraction Export group or people data with selected fields. Filter before export to extract targeted subsets (all vendors with pending credentials, all artists with incomplete forms) Feed data into external systems — print vendor email list for bulk communication, export credential names for security briefing, share artist contacts with production manager
Cross-event persistence — the organizational memory
ElementsTasksObjectives
Organization-level CRMContacts persist beyond single events Groups and people exist at the organization level, not just the event level. When you create "Summit 2027," returning vendors from "Summit 2026" are already in the CRM with their company info, contacts, and historical data Never start from scratch — the database gets richer every year. Year 3 of using Lennd, the CRM is a comprehensive vendor/artist/sponsor directory with multi-year history
Cross-event contact databaseEvery credentialed person, every event View all groups and people across all events in one unified database. Filter by event, type, status, or any field. See which contacts participated in which events Answer "which vendors have we worked with across all 12 events this year?" without opening 12 separate spreadsheets
Historical request dataPast orders, approvals, usage Each contact's profile shows their history: credential orders from past events, asset requests, catering usage, payment history. Admin can reference last year's data when planning this year Data-driven vendor management — "Acme AV used 3 golf carts and 12 credentials last year" informs this year's allocation without asking them. Reduce back-and-forth, speed up planning
Organization dashboard statsHigh-level metrics across events Organization dashboard shows aggregate stats: total events, total contacts, total revenue across all events. High-level health check on the organization's event portfolio Give organizational leadership a bird's-eye view — how many events, how many stakeholders, how much revenue, all in one glance without drilling into individual events
