Module: Integrations — ticketing, RFID, payments & data sync
Lennd's integration layer is narrow but deep — it doesn't integrate with hundreds of tools like Zapier would. Instead, it has purpose-built, bidirectional integrations with a small set of event-critical systems: ticketing platforms, RFID/access control providers, and the Stripe payment gateway. The design philosophy is: integrate deeply where data must flow automatically and in real time, rather than broadly with shallow link-outs.
Integration 1 — Ticketing platforms (Eventbrite, Front Gate Tickets)
ElementsTasksObjectives
Eventbrite connectionLennd is listed in Eventbrite App Marketplace Connect Lennd to the event's Eventbrite account. Map Lennd credential types to Eventbrite ticket types. Once connected, approved credential orders in Lennd can sync to Eventbrite as ticket issuances Eliminate double data entry — admin approves a vendor's 12 backstage passes in Lennd, and 12 corresponding tickets are issued in Eventbrite automatically, without touching Eventbrite's interface
Front Gate Tickets connectionFestival-focused ticketing partner Integrate with Front Gate Tickets for festivals that use FGT as their primary ticketing platform. Sync approved credentials to FGT fulfillment. Import barcode pools from FGT into Lennd Same goal as Eventbrite integration but for the festival vertical — Front Gate is the dominant ticketing platform for US music festivals
Credential-to-ticket syncBidirectional data flow When a credential is approved in Lennd, the corresponding ticket is created/updated in the ticketing platform. When a ticket is scanned at the gate via the ticketing platform's system, the status can sync back to Lennd's check-in records. Changes on either platform auto-sync Single source of truth across both systems — the admin doesn't need to check two dashboards to know who's been issued credentials and who's checked in
Barcode import from ticketingConnecting scan IDs to Lennd records Import barcode or ticket ID pools from the ticketing platform into Lennd. When Lennd approves a credential, the next available barcode from the imported pool is auto-assigned to that person's record Bridge the identifier gap — the ticketing platform generates barcodes, Lennd manages who gets them. The import ensures every Lennd credential maps to a scannable ticketing-platform barcode
Form-to-ticketing pipelineRequest forms feed ticket issuance Vendors or staff fill out Lennd request forms for credentials. Admin reviews and approves. Approved orders push to ticketing platform for fulfillment. The entire pipeline — form submission → admin review → approval → ticket issuance — runs without leaving Lennd until the final fulfillment step Give the admin a unified workflow — manage requests and approvals in Lennd's purpose-built interface, but leverage the ticketing platform's existing fulfillment infrastructure (email delivery, mobile tickets, box office lookup)
Integration 2 — RFID & access control (Intellitix, Elevate)
ElementsTasksObjectives
Intellitix RFID syncPrimary RFID integration partner Connect Lennd to Intellitix access control system. Approved credentials in Lennd auto-sync to Intellitix so RFID wristbands are programmed with the correct access zones. Changes in either platform sync automatically — no manual re-programming needed Close the loop between digital approval and physical access — when Lennd approves a backstage credential, the Intellitix system knows that wristband can enter backstage without any manual handoff
Elevate integrationAlternative access control partner Same bidirectional sync as Intellitix but with the Elevate platform. Credential approvals push to Elevate; access control scan events push back to Lennd Serve events that use Elevate instead of Intellitix — the admin's Lennd workflow doesn't change regardless of which RFID vendor the event uses
Access zone mappingCredential type → physical zone Map each Lennd credential type to the corresponding access zones in the RFID system. "All-Access" maps to all zones. "Vendor" maps to vendor areas + general. "VIP" maps to VIP lounge + general. This mapping is configured once and applies to all credentials of that type The access policy lives in one place (Lennd) and enforces in another (the RFID gates). The mapping ensures a vendor wristband won't open the VIP gate, even though both exist in the same RFID system
Wristband activation syncFrom credential approval to live wristband When Lennd approves a credential and assigns a barcode/RFID tag, the RFID system activates that specific tag for the mapped access zones. Revoked credentials deactivate the tag. No manual intervention at the RFID provider level Real-time access control changes — if a vendor credential is revoked at 3 PM, the wristband stops working at 3:01 PM at every RFID gate across the venue. No "we'll update it overnight"
Catering RFID check-inMeal scanning via RFID partner RFID wristbands scanned at catering stations can trigger meal claims in Lennd. The RFID system handles the physical scan, Lennd handles the meal logic (is this person approved for lunch? have they already claimed it?). Result syncs back to the RFID system for display Unified wristband experience — one tap at the gate for entry, one tap at the catering tent for lunch. The person uses one wristband for everything; two systems coordinate behind the scenes
Integration 3 — Payments (Stripe)
ElementsTasksObjectives
Stripe account connectionOrganization-level setup Connect the organization's Stripe account from the Organization settings → Billing page. One-time connection. All events under the organization use the same Stripe account for payment processing Enable all payment features (credential sales, asset billback, ticket sales, donations) with a single payment gateway connection — no per-event payment setup
PCI compliance delegationStripe handles card data All card data is processed directly by Stripe — Lennd never stores card numbers, CVVs, or sensitive payment data. Stripe's Level 1 PCI compliance covers the payment flow. SSL encryption on all data traffic The event organizer inherits Stripe's security posture without building or maintaining their own payment security infrastructure — massive compliance burden offloaded
Payment method supportCards + digital wallets via Stripe Accept credit/debit cards and Stripe-supported digital wallets (Apple Pay, Google Pay). Payments work on desktop and mobile. Stripe handles currency conversion for supported currencies (USD, CAD, AUD) Meet payers where they are — a vendor can pay their golf cart rental invoice from their phone while standing at the loading dock, using Apple Pay. No checks, no wire transfers, no "I'll pay you later"
Transaction data flowStripe → Lennd record linkage Each Stripe transaction is linked to a Lennd order, which is linked to a group, which is linked to an event. Stripe transaction ID, timestamp, amount, and status are stored in Lennd. Admin can click through from the Lennd order to the Stripe dashboard for detailed transaction info Full audit trail from the event module (what was ordered) through to the payment system (what was paid). No reconciliation gap between "what the admin sees" and "what the bank received"
Integration 4 — Data security & infrastructure
ElementsTasksObjectives
SSL/TLS encryption2048/4096-bit SHA2 public key All data traffic between users, portals, and the Lennd platform is encrypted with industry-standard SSL/TLS. Enforced on all endpoints — admin dashboard, portal access, form submissions, API calls Protect sensitive data in transit — credential names, payment information, COI documents, and personal details are never exposed to interception
Automated backupsEvery 5 hours Full database snapshots taken every 5 hours ("more than daily"). Backups stored in case of data loss or corruption. Admin does not need to trigger backups — fully automated Business continuity — even in a catastrophic data incident, the maximum data loss is 5 hours of changes. For event operations, where most data is entered days before the event, this is effectively zero risk
Data export capabilitiesCSV/Excel from every module Every module supports CSV/Excel export with configurable field selection. Credentials, catering, assets, CRM, payments — all exportable. Filtered exports for targeted data subsets Integration escape hatch — if Lennd doesn't integrate with a specific external system, the admin can always export data and import it into that system manually. No vendor lock-in on data
Custom integration requestsFor unlisted ticketing/RFID providers If an event uses a ticketing or access control provider not currently integrated (not Eventbrite, Front Gate, Intellitix, or Elevate), Lennd offers to explore custom integration development. Admin can reach out via support to discuss feasibility Expandable integration surface — the current 4 partners cover the majority of the festival/sports event market, but the platform isn't closed. New integrations are built based on customer demand
How these three infrastructure modules connect to everything above: The Event CRM is the data layer every module reads from and writes to. Team Management controls which internal staff can access which modules and whether they can approve requests. Integrations extend the platform's reach to external systems so that Lennd data (credentials, payments, access control) flows seamlessly to the ticketing platform, RFID gates, and payment processor — without the admin manually copying data between systems. Together, these three form the foundation that makes the 16 user-facing modules work as a coherent platform rather than disconnected tools.
